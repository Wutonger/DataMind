# DataMind AI 项目规划

## 1. 项目目标

DataMind AI 面向企业数据分析与数据应用场景，计划建设一套统一的智能分析工作台，围绕数据库分析、知识增强、报表输出、权限控制与执行链路追踪，形成一套可直接支撑日常分析工作的产品能力。

本文档用于明确项目建设目标、功能范围、技术方案、数据设计与验收要求，作为项目实施过程中的总体方案依据。

项目目标如下：

- 建立统一的数据源连接入口，支持多数据库连接管理与上下文切换。
- 建立统一的登录认证与权限控制能力，支持用户管理、连接授权与按用户的数据隔离。
- 建立面向数据库分析场景的智能执行能力，支持数据库工具、知识库工具与报表工具联动。
- 建立自然语言转 SQL、SQL 执行与历史管理能力，提升数据分析效率。
- 建立知识库能力，用于补充业务背景、指标口径、文档说明等非结构化信息。
- 建立图表报表与文档报告能力，支持分析结果沉淀与输出。
- 建立执行链路追踪能力，便于展示分析过程、运行步骤与关键节点。

## 2. 建设范围

### 2.1 用户与权限

计划提供统一的登录认证与权限控制模块，覆盖以下能力：

- 提供账号登录与退出能力
- 提供用户新增、编辑、删除、启用、禁用能力
- 支持管理员与普通用户角色划分
- 支持数据库连接授权
- 支持用户最近一次连接上下文保存
- 支持聊天、SQL 历史、报表等数据按用户隔离

### 2.2 数据库连接管理

计划提供统一的连接管理模块，覆盖以下能力：

- 维护数据库连接信息
- 测试连接可用性
- 按当前连接切换系统分析上下文
- 根据用户权限返回可访问的连接列表
- 为表分析、智能执行、SQL、报表、知识库提供统一连接入口

### 2.3 表结构分析

计划建设数据库结构分析能力，用于帮助用户快速理解业务库表：

- 扫描当前连接下的全部表结构
- 读取字段、类型、主键、索引等基础信息
- 生成表级中文业务描述
- 分析表之间的关联关系
- 在前端展示分析结果与处理进度

### 2.4 SQL 工作台

计划建设 SQL 工作台，作为结构化查询能力中心：

- 根据自然语言生成 SQL
- 执行 SQL 并展示结果
- 记录 SQL 生成历史
- 为智能执行、报表、图表等能力提供查询基础

### 2.5 智能执行

计划建设统一的智能执行入口，承接数据库分析主链路：

- 提供流式对话执行体验
- 根据任务自动调用数据库工具
- 根据需要调用知识库检索能力
- 根据需要调用报表保存相关工具
- 保留引用信息与执行过程

### 2.6 知识库

计划建设文档型知识库，用于补充结构化数据之外的业务语义：

- 支持文档上传、索引、检索、预览
- 文档类型包括 `PDF`、`TXT`、`Markdown`、`DOCX`
- 基于向量检索补充智能执行上下文
- 支持片段级引用与定位

### 2.7 报表中心

计划建设报表中心，用于沉淀分析结果与对外输出：

- 保存 AI 生成的图表报表
- 保存 AI 生成的 Markdown 文档报告
- 展示报表列表与内容
- 支持 PDF 下载

### 2.8 执行链路

计划建设执行链路模块，用于追踪运行过程：

- 记录智能执行、SQL、表分析、报表生成等场景的运行过程
- 保存运行记录、步骤记录、时间线记录
- 在前端页面中查看执行轨迹与关键节点

### 2.9 系统设置

计划建设系统设置模块，统一维护模型与运行参数：

- 维护语言模型配置
- 维护向量模型配置
- 维护温度等运行参数
- 对系统级配置采用管理员权限控制

## 3. 技术方案

### 3.1 前端方案

前端计划采用以下技术栈：

- Vue 3
- TypeScript
- Vite
- Naive UI
- Pinia
- ECharts

规划页面包括：

- `Login`
- `Dashboard`
- `Connections`
- `Chat`
- `SqlStudio`
- `Reports`
- `Analysis`
- `Knowledge`
- `Workflow`
- `Users`
- `Settings`

前端需要重点解决的问题包括：

- 登录态保持与权限视图切换
- 多页面下的信息密度控制
- 数据分析场景下的流式输出体验
- 执行过程可视化展示
- 图表、知识库、报表等模块的一致化交互

### 3.2 后端方案

后端计划采用以下技术栈：

- Spring Boot
- Spring AI
- Spring AI Alibaba
- Spring Data JPA
- MySQL
- Sa-Token
- MCP

后端模块规划如下：

- `data-analysis-common`
- `data-analysis-mcp-server`
- `data-analysis-mcp-client`
- `data-analysis-skills`
- `data-analysis-agent`
- `data-analysis-core`
- `data-analysis-web`

各模块职责规划如下：

- `data-analysis-common`：承载公共常量、工具类、基础模型
- `data-analysis-mcp-server`：对外暴露数据库工具能力
- `data-analysis-mcp-client`：封装 MCP Client 接入逻辑
- `data-analysis-skills`：承载 Skill 资源与保存型工具
- `data-analysis-agent`：承载 Agent 编排、工具拦截、执行链路追踪
- `data-analysis-core`：承载连接、聊天、知识库、报表、表分析等核心业务
- `data-analysis-web`：承载 REST API 与应用启动入口

### 3.3 Agent 与 Tool 组织方式

项目计划采用统一 Agent 编排方案，核心设计如下：

- 由统一 Agent 编排层处理智能执行、SQL、报表等主链路
- 数据库能力通过 MCP Server 暴露为标准工具
- 业务约束与专业提示通过 Skill 资源组织
- 运行过程通过 Workflow 相关表进行追踪

Skill 资源规划包括：

- `knowledge-grounding`
- `artifact-generation`
- `insight-discovery`

## 4. 数据设计

项目计划基于 MySQL 保存平台元数据与运行记录。核心数据表规划如下：

- `sys_user`
- `connection_user_access`
- `connections`
- `table_metadata`
- `chat_sessions`
- `knowledge_documents`
- `document_chunks`
- `reports`
- `sql_history`
- `workflow_runs`
- `workflow_steps`
- `workflow_timeline`
- `app_config`

数据库初始化脚本规划放置于：

- `sql/init.sql`

数据设计原则如下：

- 登录身份、角色与授权范围独立建模
- 连接、分析、知识库、报表、链路等核心领域数据独立建模
- 运行过程与最终结果分层记录，便于查询与展示
- 当前连接上下文贯穿智能执行、SQL、知识库和报表能力
- 用户私有数据统一通过 `user_id` 归属
- 管理员配置能力与普通用户使用能力在页面和接口层同时控制

## 5. 核心流程设计

### 5.1 登录与授权链路

1. 用户提交用户名和密码
2. 系统校验账号状态与密码摘要
3. 生成登录态并返回当前用户信息
4. 系统读取用户最近一次连接与可访问连接范围
5. 管理员可进入用户与权限模块维护账号和连接授权

### 5.2 智能执行链路

1. 用户提交任务
2. 系统绑定当前会话与连接上下文
3. Agent 根据任务选择数据库工具、知识库工具或报表保存工具
4. 过程通过 SSE 流式返回给前端
5. 结果、步骤和引用信息写入历史记录
6. Workflow 记录整条执行链路

### 5.3 SQL 生成链路

1. 用户输入自然语言需求
2. Agent 结合 schema 和知识上下文生成 SQL
3. 返回 SQL 结果
4. 记录生成历史与执行轨迹

### 5.4 表结构分析链路

1. 扫描全库元数据
2. 生成中文业务描述
3. 分析全库表关系
4. 回写分析结果
5. 记录分析过程与步骤

### 5.5 报表生成链路

1. 用户提出图表或文档报告需求
2. Agent 结合 schema、知识库和查询结果组织产物
3. 通过保存工具写入报表中心
4. Workflow 记录过程与结果

### 5.6 知识库链路

1. 上传文档并保存源文件
2. 读取文档内容并切块
3. 生成向量并入库
4. 在智能执行中按需检索
5. 前端支持片段预览和引用定位

## 6. 建设要求

- 知识库文档类型支持 `PDF`、`TXT`、`Markdown`、`DOCX`
- 执行链路采用运行记录、步骤记录、时间线记录三层结构持久化
- 智能执行、SQL、知识库、报表等模块共享统一连接上下文
- 普通用户仅可访问被授权连接，管理员具备全量连接管理与系统配置权限
- 聊天、SQL 历史、报表等用户数据需按账号隔离保存
- 前后端页面风格保持统一，重点场景具备清晰的过程反馈

## 7. 验收目标

- 用户可以完成登录并进入与自身权限相匹配的功能页面
- 管理员可以维护用户并为普通用户授权数据库连接
- 用户可以维护数据库连接并切换当前分析上下文
- 用户可以完成全库表分析并查看结果
- 用户可以通过自然语言生成并执行 SQL
- 用户可以在智能执行中完成数据库分析与知识增强任务
- 用户可以将图表或文档保存到报表中心
- 用户可以在知识库中上传文档、检索内容、预览片段
- 用户可以在执行链路页面查看关键运行过程
- 用户可以在系统设置中维护模型配置
